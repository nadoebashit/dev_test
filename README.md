Проект разделен на две независимые части микросервисы, которые общаются друг с другом по сети. Обе части упакованы в Docker, так что вам не придется возиться с установкой языков и прочими системными настройками.

### Фронтенд (Интерфейс)

- **Стек:** React 18, Next.js 14, TailwindCSS.
- **Состояния:** За данные на клиенте отвечает легковесный стейт-менеджер **Zustand**. Он сохраняет погоду и передает ее на страницу с результатами без лишних перезагрузок.
- **Внешний вид:** Интерфейс не только адаптирован под мобилки,  страница с температурой сама меняет свой цвет в зависимости от градусов за окном.
- **Слой-клиент к API:** Вся работа с бэкендом вынесена в аккуратный файл конфигурации `src/lib/api.ts`.

### Бэкенд (Логика и логирование)

- **Стек:** Python 3.11, FastAPI, Pydantic, SQLAlchemy 2 (async), Postgres.
- **Абсолютная асинхронность:** Вся база данных и внешние веб-запросы (через библиотеку `httpx`) работают асинхронно, так что сервер может обрабатывать миллионы конкурентных запросов без блокировок.
- **Фоновые обновления (Cron):** Под капотом тикает периодическая задача (через пакет `APScheduler`). Раз в 30 минут она сама проверяет все города в базе и обновляет для них свежую температуру!
- **Источники данных:** Бэкенд забирает данные об атмосферном давлении, влажности, координатах и температуре из открытого сурса Open-Meteo.

## Запуск проекта

Вам понадобится только установленный Docker!

1. Склонируйте репозиторий:

   ```bash
   git clone <URL>
   cd dev_test
   ```

2. Запустите все контейнеры одной простой командой.:

   ```bash
   docker-compose up --build -d
   ```

3. Как только контейнеры запустятся, не забудте накатит миграции на базу данных, чтобы создались нужные таблички:
   ```bash
   docker-compose exec backend alembic upgrade head
   ```

- Фронтенд работает тут: [http://localhost:3000](http://localhost:3000)
- Автодокументация API (Swagger) тут: [http://localhost:8000/docs](http://localhost:8000/docs)

## Тесты

Ключевая логика API покрыта unit-тестами. Библиотека `pytest` тестирует наши эндпоинты, мокая внешние HTTP-запросы и используя временную базу данных (`aiosqlite` в оперативной памяти).

Чтобы запустить тесты, просто выполните эту команду на фоне запущенного контейнера `backend`:

```bash
docker-compose exec backend pytest tests/ -v
```
